JSP:

1. 指令
    * 作用 : 用于配置jsp页面,导入资源文件
    * 格式 :
        <%@ 指令名称 属性名1=属性值1 属性名2= 属性值2 ... %>
    * 分类 : 
        1. page      : 配置jsp页面用的
            * contentType  : 等同于response. setContentType()
                1. 设置响应体的mime类型以及字符集
                2. 设置当前jsp页面的编码(只能是高级的IDE才能生效)
            * import  : 导包
            * errorPage : 当前页面发生异常后,会自动跳转到指定的错误页面
            * isErrorPage : 标识当前也是是否为错误页面
                * true : 是, 可以使用exception 对象
                * false : 否 , 默认值,不可以使用内置对象
        2. include : 页面包含的,导入页面的资源文件
        3. taglib : 导入资源
            * prefix 自定义的 
2. 注释
    1.  html注释
    2.  jsp注释
        <%-- --%>  : 可以注释所有
3. 内置对象

MVC : 开发模式

1. jsp演变历史
    1. 早期只有servlet,只能使用response输出标签数据,非常麻烦
    2. 后来有了jsp,简化了Servlet的开发,如果过度使用jsp,在jsp中即写大量的java代码,又写HTML标签,难以维护,难于分工
    3. 再后来,javaweb开发,借鉴mvc开发模式,是的程序的设计更加合理性

2. MVC :
    1.  M : model,模型   **JavaBean**
        * 完成具体的业务操作,封装对象
    2.  V : view ,视图        **JSP**
        * 展示数据
    3.  C : controller, 控制器     **Servlet**
        * 获取用户的输入
        * 调用数据
        * 将数据交给视图进行展示


    * 优点 
        1. 耦合性低,方便分工合作,方便维护
        2. 重用性高

    * 缺点
        1. 使得项目架构变得复杂,对开发人员要求更高

EL表达式

1. 概念 : EXpression Language 表达式语言
2.  作用 : 替换和简化jsp页面中java代码的编写
3.  语法 : ${表达式}
4.  注意 : 
    * jsp是默认支持EL表达式的. 如果要忽略EL表达式
        1. 是指jsp中page指令中: isELIgnored = "true" 忽略当前页面中的所有EL表达式
        2. \ 转义

5. 使用 :
    1. 运算 :
        1. 算数运算符  除法/(div) 取模%(mod)
        2. 比较运算符
        3. 逻辑运算符
        4. 空运算符 : empty :  
             * 用于判断字符串,集合数组对象是否为null并且长度是否为0
             * ${empty list}  :判断字符串,数组,集合对象是否为null或者长度为0
             * ${not empty str } : 表示判断字符串,集合,数组对象是否为null并且长度大于0

    2. 获取值 
        1.  EL表达式只能从域对象中获取值
        2. 语法 :
            1. ${域名称.键名} :从指定域中获取指定值
                * 域名称
                    1. pageScope   -->pageContext
                    2. requestScope -->request
                    3. sessionScope --> session
                    4. applicationScope -->application(ServletContext)

                * 举例 :在request域中存储了name=张三
                * 获取 : ${requestScope.name}
                * ${键名} : 表示依次从最小的域中查找是否又该键对应的值,知道找到为止.

        3. 获取对象 , list 集合 , Map集合
            1. 对象 : ${域名称.键名称.属性名}
                本质上回去调用对象的getter方法
            2. List集合 :${域名称.键名[索引]}
            3. Map集合 :${域名称.键名.key名称}

    3. 隐式对象 :

JSTL

1. 概念 : JavaServer Pages Tag Library JSP标准标签库
    * 是由Apache组织提供的

2. 作用 : 用于简化和替换jsp页面上的java代码


3. 使用步骤 :
    1. 导入jar包
    2. 引入标签库 :taglib指令  : <%@ taglib %>
    3. 使用标签

4. 常用的JSTL标签
    1.  if        :相当于java代码的if语句
            if标签体内必须加test
                作用 : 接收boolean表达式

                用法 : 一般情况下test结合EL一起使用

                注意 : 这个if标签没又else情况,如果想要的话再后面再写一个if即可
    2.   choose    :相当于switch
            *完成数值编号对应星期几案例
                     1. choose --> switch
                     2. when --> case
                     3. otherwise --> default
    3.   foreach   :相当于for
            * for循环的用途
                    1. 执行重复的操作
                        for(int i = 0;i<10; i++){

                        }
                        标签 : begin end var step varStatus
                                开始  结束 变量  步长   循环状体的对象
                    2. 遍历容器
                        List<User> list
                        for(User user : list){

                        }
                        标签:
                            items : 容器对象
                            var : 容器的临时变量
                            varStatus : 循环状态的对象
                                index : 对应索引
                                count : 循环次数
5. 练习 :
    * 需求 :再request域中有一个存在User对象的List集合.需要使用jstl+el将list集合展示到jsp页面的table表格中
        <%
            List list = new ArrayList();
            list.add(new User("张三",new Date(),10));
            list.add(new User("李四",new Date(),10));
            list.add(new User("王五",new Date(),10));
            list.add(new User("赵六",new Date(),10));

            request.setAttribute("list",list);
        %>
        <table border="1" width="500">
            <th>编号</th>
            <th>姓名</th>
            <th>生日</th>
            <th>年龄</th>

        <c:forEach items="${list}" var="user" varStatus="s">
            <tr>
                <td>${s.count}</td>
                <td>${user.name}</td>
                <td>${user.date}</td>
                <td>${user.age}</td>
            </tr>
        </c:forEach>
        </table>

三层架构 : 软件设计架构

1. 界面层(表示层)  : 用户看的到的界面.用户可以通过界面上的组件和服务器交互
2. 业务逻辑层 :处理业务逻辑的
3. 数据访问层 : 操作数据存储文件的
